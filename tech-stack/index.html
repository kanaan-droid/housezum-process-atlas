Non-negotiable architectural rules

▸ Thin route handlers

API routes: auth + call service + return. Business logic lives in service classes only. Required for Lambda migration compatibility.

▸ No Vercel coupling

No @vercel/kv or @vercel/edge-config. Every new Vercel cron = ROADMAP aws-migration tag. If it only works on Vercel, flag before adding.

▸ Storage paths only

Never store absolute URLs. Always storage_path column. Maps cleanly to S3 on migration. No Supabase-specific URL construction in components.

▸ Country-agnostic

No hardcoded AED, RERA, or single-country values. All market-specific strings in lookup maps. UAE = fallback only, not primary path.

▸ org_id on everything

Every table row has org_id. RLS enforced at DB level. Never filter by org in application code as primary security — DB enforces it.

▸ AI wrapper only

All model calls through central ai_service — never direct Anthropic/OpenAI SDK calls in components. Model routing, cost control, logging in one place.

▸ Behavior before data

Ship features that create agent behavior first. Capture signal from that behavior second. Don't build logging before there's anything to log.

▸ Async always

Planning, scoring, and AI generation run on schedules or in background. Never block a user action on an AI call. Priority queue: real-time > batch.

▸ Outcomes or nothing

A system that tracks activity but not outcomes builds no real ML advantage. Capture deal won/lost/reason from day one.
