1. Deal Market Binding & Workflow Auto-Attach (BL-169)

Shipped 2026-06-06. Documents the mechanism that runs when a deal is created.

What happens when a deal is created

When an admin or agent creates a deal (the New Deal modal), the system now binds the deal to a market and attaches a workflow pipeline automatically — no manual selection.

1. Market country is derived (server-side).

  • If a property is linked, the deal's market country comes from that property (the property is the source of truth for where the deal happens).
  • If no property is linked, it falls back to the agency's country.
  • The value is stored as an ISO-2 code (CH, FR, IT, DE, AE) on deals.market_country_code. This is the field the compliance engine uses to decide which country's rules apply.

2. A workflow pipeline is auto-attached.

  • The system finds the matching federal workflow template for the deal's country and type (Sale or Rental) and attaches it as the deal's pipeline (workflow_instance).
  • "Federal" = the country-wide template (e.g. Switzerland Property Sale), not a regional/cantonal variant.
  • There is no template picker — the pipeline is chosen by country + type. (Agencies customise pipelines once, via Clone on the Workflows page; deals just use the right one.)
  • The deal-detail page then shows the real pipeline stages (e.g. the Swiss Sale stages) instead of a generic stage bar.

What this enables

With a market country on the deal, the compliance engine evaluates the correct country's rules for that deal. Before this, deals created in the app had no country and so matched no country rules (see the finding note) — the country compliance layer was effectively dark for user-created deals.

Current limits (by design, this release)

  • Region/canton is not yet derived. market_region_code stays empty for now (region-specific rules are a fast-follow — tracked as BL-170). Country-level compliance is fully active.
  • Germany: German deals do auto-attach a pipeline — the federal Germany Sale (BL-164) / Germany Rental (BL-164) templates exist and are active. A DE Sale deal binds market_country_code='DE', attaches the Germany Sale pipeline, and fires DE compliance rules (verified live 2026-06-07, 18 rules vs 0 baseline). The only DE gap is that DE templates can't be authored/cloned in-app yet (Create-Template dropdown → BL-171) — that doesn't affect deals.
  • Other markets: Switzerland, France, Germany, Italy attach pipelines today. Italy Rental and UAE Rental templates don't exist yet, so those combinations attach no pipeline (deal still valid).
  • No automatic backfill: existing older deals created before this release keep their previous (empty) market/pipeline; only new deals are affected.

Verifying (Scenario A)

Create a Sale deal on a Swiss property → the deal should have market_country_code = CH, a workflow_instance_id, the Switzerland Property Sale pipeline on the detail page, and country compliance rules available for it. (Full acceptance checks A1–A7 in the spec.)
