1. What it does (plain English)

Many of HouseZoom's compliance rules are regional, not just national — Swiss rules differ by canton, German rules by Bundesland. For those rules to fire, the system has to know which sub-national region a property sits in.

We do not ask the agent to pick a region from a dropdown. Instead, the agent types the one thing they already know — the postal code — and HouseZoom works out the region automatically. Type 8001 on a Zürich property and the system silently derives canton Zürich (CH-ZH); that region then flows into any deal on the property, which switches on the correct cantonal rulebook.

In short: the agent types a postal code; HouseZoom figures out the legal region.

3. How it works (user-facing flow)

  1. Agent opens a property's Edit screen, expands Location, and types a postal code.
  2. On Save, HouseZoom looks the postal code up against the country's official postal→region table and stores the matching region on the property.
  3. If a deal is later created on that property, the deal inherits the region, and the correct regional rulebook (and workflow) switches on automatically.

The agent does nothing special — they type a postal code and save. Everything downstream follows.

Worked example (live, Zürich Premium Realty):

  • Agent sets Postal Code 8001 on Limmatquai 56 → region resolves to CH-ZH (canton Zürich).
  • A new deal on that property binds to CH-ZH and attaches the Swiss Sale workflow (Lex Koller, notarial deed, land registry stages).
  • Change the postal code to 1201 (Geneva) and the region follows to CH-GE — proving the postal code, not the address text, drives the region.

4. The three location fields — don't confuse them

This is the most common source of confusion. There are three distinct fields:

FieldWhat it isWho sets itExample
Postal CodeThe agent's typed input. Core property data.Agent (UI)8001
Region / CityA free-text label for display. Cosmetic.Agent (UI)Kanton Zürich
region_code (engine)The derived ISO 3166-2 code that rules match on.System (derived)CH-ZH

Postal Code and region_code are the load-bearing pair: the agent types the first, the system derives the second. Region / City is just a human-readable label and has no effect on compliance — it can say anything and the engine ignores it.

5. What happens when the postal code can't be resolved

  • Unknown / invalid postal code (e.g. a typo, or a country with no data yet): the postal code is still saved (it's valid data), but no region is derived — the system never guesses. The deal then runs on national rules only (correct, safe degradation — it never blocks a transaction).
  • Country not yet seeded (today: Germany, France, Italy, UAE): same behavior — postal saved, region not derived, national rules apply. Regional derivation switches on for those countries the moment their postal data is loaded (tracked under Data Environments → Geo Data Seeding).
