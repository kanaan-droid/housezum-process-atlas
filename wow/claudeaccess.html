How Claude.ai web delivers code to Will

  1. Generate file with create_file (or str_replace/bash_tool) in /home/claude/.
  2. Call present_files with the path — this copies it to /mnt/user-data/outputs/ and surfaces it in the chat for Will to download.
  3. Will manually moves it to ~/Downloads/claude/ and from there to ~/Apps/housezum-app/.

What Claude.ai web is BEST FOR

  • Spec writing, ADR cross-validation, architectural diagrams
  • MCP browser verification (Chrome MCP) — driving production app to verify features. This was previously assumed to require Claude Desktop. It does not.
  • Generating CC prompt files (.md instructions for CC to execute)
  • Generating SQL migration scripts (for Will to run manually in Supabase SQL editor)
  • Cross-validation conversations against ChatGPT / Gemini output (Will pastes their text in)

What Claude.ai web is NOT for

  • Running migrations (CC also can't — Will runs manually)
  • Editing files in the repo directly (use CC for that)
  • Long-running watch/serve processes (no persistent runtime)

Claude Desktop (Will's Mac app) — what's different

  • Has filesystem MCP → can read/write Will's repo, run git, run npm scripts
  • Has Chrome MCP via extension pairing
  • No persistent memory between conversations (no userMemories)
  • Best for: full-loop dev tasks where Claude needs to read repo + run scripts + drive browser, AND Will wants to watch each step

When to prefer Claude Desktop over Claude.ai web

  • Heavy file-editing across many repo files (Claude.ai's download-from-/home/claude/ flow gets tedious past ~3-4 files)
  • Tasks that need git operations Claude itself executes
  • When Will wants to keep his fingers off the keyboard for an extended session

When to prefer Claude.ai web over Claude Desktop

  • Anything benefiting from persistent memory of Will/HouseZoom context
  • Spec/prompt writing (no repo writes needed; preserves clean repo state)
  • Browser verification IS available here (proven 2026-05-21) — was wrongly believed to be Desktop-only
  • Cross-validation work (paste-in friendly)

Claude Code (CC) — what's different

  • Native filesystem + bash, no MCP needed for either
  • Designed for terminal/repo work
  • No browser access (no Chrome MCP)
  • No web search
  • Will dispatches CC via prompt files Will places in ~/Downloads/claude/ then types: "Read and execute: /Users/williamkanaan/Downloads/claude/[filename].md"
  • CC never runs SQL migrations — Will runs them manually in Supabase SQL Editor

CC's strengths

  • Fast file editing at scale
  • Running test suites, type checks, linters
  • Git operations
  • Executing the markdown prompt files Claude.ai web generates

CC's weaknesses (do not dispatch CC for)

  • MCP browser verification — Phase 3 walks are NOT CC work
  • Anything requiring web fetch beyond WebFetch tool
  • Tasks where reading external context (web search results, latest docs) matters
  • Tasks where it would need to ask Will follow-up questions mid-run (CC tends to plow forward; better to script the decision tree in the prompt)
