Absorbed tickets

  • BL-115a — Schema fallback classifier tightened with ~25 synonym mappings — files: src/services/ai/property-schema-fallback.service.ts

After BL-115 shipped, the Haiku column classifier missed common natural-language phrasings ("how big is the property" → didn't map to size_sqm, "when was it built" → didn't map to year_built, "lease duration" → didn't map to lease_term_months). D-198 added ~25 synonym mappings to CLASSIFIER_SYSTEM covering size/area, year/age, and lease-related phrasings. Live-verified: "how big is the property" now returns size_sqm via fallback. Pattern: synonym maps live in the classifier prompt, not in the matcher — the matcher's job is intent+context, the fallback's job is column-resolution. When users surface a missed natural phrasing, the fix is a synonym entry in CLASSIFIER_SYSTEM, not a new keyword in the matcher.

Absorbed tickets

  • BL-104 — Dashboard Q&A Context Resolution (deferred — data sieve cannot ground without property/deal context) — files: dashboard AssistantBar (entry point); resolution would touch intent-matcher.service.ts, context-assembler.ts — [deferred until BL-60 v1 real usage data]

When an agent asks a question from the dashboard AssistantBar (not from Viewing Mode), there is no property_id or deal_id in scope. The Data Sieve hard-gates on null/NYT/stale, so most ground-truth answers can't be produced — the engine would have to fall back to non-data-grounded responses, which violates the engine's "ground truth or refuse" invariant. Four v2 options on the table: (a) entity-reference detection ("the Burj Khalifa one"), (b) active-deal selection, (c) dropdown picker, (d) non-data-grounded-only fallback. Decision deferred until real usage data from BL-60 v1 shows which option agents actually want. Don't pre-build entity-reference NLU without evidence.

Operational concerns

  • BL-112 — Auto-focus QaInput on mount so keyboard pops up immediately — files: src/features/ai-qa/components/QaInput.tsx

Mobile-first UX rule for the Q&A panel: when the panel slides up, the input must auto-focus so the keyboard opens immediately, saving one tap. Implemented via a ref + useEffect that calls .focus() on mount. Trivial in code, important for the agent-in-the-field experience that drives all Q&A panel decisions.

  • BL-117 — Rebuild matcher cache with tagged corpus — files: matcher cache rebuild script

The intent matcher uses a precomputed cache of qa_knowledge_base rows tagged with derived keywords (the keyword short-circuit at keyword-matcher.ts short-circuits <200ms when an exact tag matches). When the corpus changes (BL-60 v33 reseeded, BL-115b added 20 rows, etc.) the cache must be rebuilt against the new tagged corpus. This is operational maintenance, not a code-time concern: triggered manually or via cron after corpus changes. If keyword short-circuit hit-rate drops noticeably in observability, suspect a stale cache against a freshly-reseeded corpus.
