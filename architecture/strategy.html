AI Architecture

v1.1 · April 2026 · cross-validated

System design, data moat, and AI orchestration strategy for HouseZoom.ai. Updated after cross-validation with Gemini and ChatGPT.

Three-layer agentic orchestration: Signal Agents detect, the Intelligence Engine scores, the Planning Agent acts. Multi-model routing underneath.

North star

HouseZoom is not an AI company that sells real estate software. It's a real estate workflow company with proprietary AI built on CRM ground truth.

The moat is not the models. The moat is accumulated workflow, compliance, correction, and communication data that compounds over time. A standalone LLM wrapper has none of it.

Existential threat

⚠ Portals moving upstream

PropertyFinder, Bayut, and SeLoger already own distribution AND data exhaust. They see: lead → click → contact → conversion. We only see: lead → agent actions. If they ship "AI CRM-lite," the top-of-funnel moat compresses fast.

Defense: our moat must be inside the deal — compliance, document workflow, multi-party signing, deal orchestration. Portals cannot easily follow here. Double down on this layer.

Non-negotiable architectural rules

▸ Thin route handlers — API routes: auth + call service + return. Business logic lives in service classes only. Required for Lambda migration compatibility.

▸ No Vercel coupling — No @vercel/kv or @vercel/edge-config. Every new Vercel cron = ROADMAP aws-migration tag. If it only works on Vercel, flag before adding.

▸ Storage paths only — Never store absolute URLs. Always storage_path column. Maps cleanly to S3 on migration. No Supabase-specific URL construction in components.

▸ Country-agnostic — No hardcoded AED, RERA, or single-country values. All market-specific strings in lookup maps. UAE = fallback only, not primary path.

▸ orgid on everything — Every table row has orgid. RLS enforced at DB level. Never filter by org in application code as primary security — DB enforces it.

▸ AI wrapper only — All model calls through central ai_service — never direct Anthropic/OpenAI SDK calls in components. Model routing, cost control, logging in one place.

▸ Behavior before data — Ship features that create agent behavior first. Capture signal from that behavior second. Don't build logging before there's anything to log.

▸ Async always — Planning, scoring, and AI generation run on schedules or in background. Never block a user action on an AI call. Priority queue: real-time > batch.

▸ Outcomes or nothing — A system that tracks activity but not outcomes builds no real ML advantage. Capture deal won/lost/reason from day one.
