AI Architecture

System design, data moat, and AI orchestration strategy for HouseZoom.ai. Updated after cross-validation with Gemini and ChatGPT.

North star

HouseZoom is not an AI company that sells real estate software.
It's a real estate workflow company with proprietary AI built on CRM ground truth.

The moat is not the models. The moat is accumulated workflow, compliance, correction, and communication data that compounds over time. A standalone LLM wrapper has none of it.

AI pipeline — how a signal becomes an action

1 · Detect

Raw event arrives from portal, Gmail, WhatsApp, or CRM action.

→ inbound_signals
→ entity resolution
→ lead_sessions

2 · Score

Intelligence engine scores the signal: urgency, value, confidence.

→ P = U×V×C
→ decay applied
→ intel_cards

3 · Plan

Planning agent assembles ranked daily action list. Specific, contextual, actionable.

→ daily 7am batch
→ real-time triggers
→ dismiss/snooze loop

4 · Learn

Agent action (or inaction) feeds back. Corrections captured. Model improves.

→ ai_feedback
→ delta capture
→ ranking update
Data moat — status
STRONG

Compliance graph

Country × property type × deal stage → required docs and checks. High value, expensive to maintain, hard to replicate.

STRONG

Deal graph

Who is involved, in what role, at what stage, with what documents. Real durations per stage — no public dataset has this.

BUILDING

Communication graph

Who talks to whom, how fast, in what language. Captured but not yet scored. Strongest latent asset — nobody in RE uses it properly.

BUILDING

Agent correction data

Delta between AI draft and what agent actually sent. Delta capture table not yet built — every day without it loses signal.

FUTURE

Deal outcomes

Won/lost reasons, time-to-close per stage. Ground truth for all future ML. Nothing works without this — must be captured from day one.

FUTURE

Cross-org intelligence

"Marina deals close 18% faster with X pricing." Anonymized benchmarks across agencies. Network effect moat — requires scale.

Existential threat
⚠

Portals moving upstream

PropertyFinder, Bayut, and SeLoger already own distribution AND data exhaust. They see: lead → click → contact → conversion. We only see: lead → agent actions. If they ship "AI CRM-lite," the top-of-funnel moat compresses fast.

Defense: our moat must be inside the deal — compliance, document workflow, multi-party signing, deal orchestration. Portals cannot easily follow here. Double down on this layer.
