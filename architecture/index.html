HouseZoom system architecture — four tiers from client surfaces to external integrations.

What HouseZoom is (one paragraph)

HouseZoom.ai is an AI-powered, multi-tenant real estate CRM for agents. Multi-market: UAE (primary), France, Switzerland, Italy. Built on Next.js 14 (App Router) + TypeScript + Tailwind, with Supabase (Postgres + Storage + Auth + RLS) as the backend, deployed on Vercel. iOS distribution via Capacitor in remote URL mode (loads the live Vercel build). AI features run on Anthropic Claude (Sonnet + Haiku) via a central ai_service wrapper. The product positions itself as a real-estate workflow company with proprietary AI built on CRM ground truth — not an AI company that sells real-estate software (see STRATEGY.md).

Invariants (the rules that never change)

These come from WAYS-OF-WORKING.md (the canonical home for non-negotiable rules). Changes belong there first; this section mirrors.

  • org_id on everything. Every table has org_id. RLS enforced at the DB level. Never filter by org in application code as primary security — the DB enforces it.
  • No hardcoded market-specific values. Currency codes, country names, license labels, API URLs — all via lookup maps or useOrg(). UAE values are fallback only, never the primary path.
  • Service-layer DB access. Components and hooks never import supabase directly. All DB access goes through service files in src/features/{domain}/services/ or src/services/.
  • Thin route handlers. API routes do auth + service call + response. Business logic lives in service classes — required for AWS Lambda migration compatibility.
  • All AI calls go through the central wrapper. No direct Anthropic/OpenAI SDK calls in components. Model routing, cost control, and logging in one place.
  • Fail-open for AI / cache / external services. If Claude or Upstash or Twilio is down, the user-facing surface degrades gracefully — never blocks an agent action.
  • Storage paths only. Never store absolute URLs in DB. Always storage_path columns. Maps cleanly to S3 on AWS migration.
