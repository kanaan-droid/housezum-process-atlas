Geo Data Seeding — Coverage

Status: Living document. Updated whenever postal→region data is seeded for a country. Version: v0.1 Last updated: 2026-06-08 (CH seeded — BL-POSTAL-TO-REGION / D-397) Maintainer: Will + Claude

1. What this tracks

The geo-resolution capability (postal code → ISO 3166-2 region) is country-agnostic — see Geo & Locations for how it works. This page tracks the other half: which countries actually have postal data loaded into the postal_regions table on the live environment.

A country with no seeded data degrades safely — postal codes are still captured, but no region is derived, so deals run on national rules only. Derivation switches on for a country the moment its data lands here.

3. Switzerland — detail (seeded)

  • Source: swisstopo Official Directory of Towns and Cities (canton per NPA), obtained via the zauberware postal-codes-json-xml-csv mirror (CC-BY 4.0, GeoNames-derived).
  • Shape: 3,362 distinct 4-digit NPAs, all 26 cantons, 0 blank cantons.
  • Ambiguous: 21 multi-canton NPAs flagged is_ambiguous = true; primary canton cross-validated against ChatGPT + Gemini.
  • Pending: 5 NPAs (1534, 1565, 1595, 2333, 9532) carry best-effort primaries awaiting verification against the official Swiss Post PLZ perimeter dataset. All flagged ambiguous; safe degradation.
  • Seed files: supabase/seeds/postal_regions_ch.sql (base) + ambiguous primary patch. Generated by scripts/generate-postal-regions-seed.mjs (reproducible from source).

4. Bringing a country online

  1. Source an authoritative, reuse-licensed postal→region dataset.
  2. Cross-validate border-straddling codes; flag is_ambiguous.
  3. Generate + run an idempotent seed for that country_code (Will runs).
  4. Verify shape (row count, region coverage, 0 blanks) before marking seeded.
  5. Update this matrix.
